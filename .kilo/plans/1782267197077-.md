# Plan: Integrate apk-tweak into Revanced-auto

## Goal
Archive [Ven0m0/apk-tweak](https://github.com/Ven0m0/apk-tweak) and merge all its features into Revanced-auto as a unified build pipeline.

---

## Feature Inventory (apk-tweak → Revanced-auto)

| # | Feature | Source (apk-tweak) | Integration Target | Priority |
|---|---------|-------------------|-------------------|----------|
| 1 | **Media Optimizer** | `rvp/engines/media_optimizer.py` | `scripts/builder/media_optimizer.py` | High |
| 2 | **APK Optimizer** (debug strip, native lib strip, manifest minimize, locale filter) | `rvp/engines/optimizer.py` | `scripts/builder/apk_optimizer.py` | High |
| 3 | **String Cleaner** (unused string removal via apktool) | `rvp/engines/string_cleaner.py` | `scripts/builder/string_cleaner.py` | Medium |
| 4 | **DTL-X Engine** (ad/tracker removal, security bypasses, code optimization) | `rvp/engines/dtlx.py` | `scripts/builder/dtlx.py` | Medium |
| 5 | **LSPatch Engine** (alternative patching method) | `rvp/engines/lspatch.py` | `scripts/builder/lspatch.py` | Medium |
| 6 | **RKPairip Engine** (advanced decompile/rebuild, string recovery) | `rvp/engines/rkpairip.py` | `scripts/builder/rkpairip.py` | Low |

| 9 | **Ad-blocking patterns** | `rvp/ad_patterns.py` | `scripts/utils/ad_patterns.py` | Low |

## Architecture Decision

**Approach: Port engines as optional per-app build stages within Revanced-auto's existing pipeline.**

Revanced-auto's pipeline: `config → version check → download → patch → sign → output`

New pipeline with apk-tweak features:
```
config → version check → download → [pre-patch engines] → patch → [post-patch engines] → sign → output
```

- **Pre-patch engines** (run before ReVanced patching): LSPatch, RKPairip, DTL-X
- **Post-patch engines** (run after ReVanced patching, before signing): Media Optimizer, APK Optimizer, String Cleaner
- **Plugin hooks**: Fire at each stage boundary (pre_pipeline, pre_engine, post_engine, post_pipeline)

Each engine is **opt-in per app** via `config.toml` settings. Disabled by default to preserve existing behavior.

## Implementation Phases

### Phase 1: Core Infrastructure (Foundation)

#### 1.1 Create engine base module
- **File**: `scripts/builder/engines/__init__.py`
- Define `EngineContext` dataclass (wraps `AppBuildContext` fields needed by engines)
- Define `EngineResult` dataclass (success, output_apk, metadata, error)
- Define `Engine` protocol: `run(ctx: EngineContext) -> EngineResult`
- Define engine stage enum: `PRE_PATCH`, `POST_PATCH`

#### 1.2 Extend configuration
- **File**: `scripts/builder/config.py`
- Add to `GlobalConfig`:
  ```python
  # Engine toggles (global defaults)
  enable_media_optimizer: bool = False
  enable_apk_optimizer: bool = False
  enable_string_cleaner: bool = False
  enable_dtlx: bool = False
  enable_lspatch: bool = False
  enable_rkpairip: bool = False
  enable_whatsapp_patcher: bool = False
  # Engine tool paths
  dtlx_path: str = ""
  lspatch_jar: str = ""
  rkpairip_path: str = ""
  ```
- Add to `AppConfig`:
  ```python
  # Per-app engine overrides
  enable_media_optimizer: bool | None = None  # None = inherit global
  enable_apk_optimizer: bool | None = None
  enable_string_cleaner: bool | None = None
  enable_dtlx: bool | None = None
  enable_lspatch: bool | None = None
  lspatch_mode: str = "complement"  # "complement" or "alternative"
  # Per-app engine options
  media_optimizer_options: dict = field(default_factory=dict)
  apk_optimizer_options: dict = field(default_factory=dict)
  dtlx_options: dict = field(default_factory=dict)
  lspatch_options: dict = field(default_factory=dict)
  ```
- Update `from_dict()` methods for both dataclasses

#### 1.3 Extend config.toml
```toml
# Global engine defaults
enable-media-optimizer = false
enable-apk-optimizer = false
enable-string-cleaner = false
enable-dtlx = false

# Per-app engine config
[YouTube-Morphed]

# ... existing settings ...
enable-media-optimizer = true
enable-apk-optimizer = true

[YouTube-Morphed.media-optimizer]

optimize-audio = true
target-dpi = "xxhdpi"
png-quality = "65-80"
jpeg-quality = 85

[YouTube-Morphed.apk-optimizer]
remove-debug-symbols = true
minimize-manifest = true
optimize-resources = true
keep-locales = ["en"]
strip-native-libs = true

[YouTube-Morphed.dtlx]
rmads1 = true
rmads2 = true
rmtrackers = true
sslbypass = false
```

#### 1.4 Integrate engine runner into build pipeline
- **File**: `scripts/builder/app_processor.py`
- Add `_run_pre_patch_engines(context)` method — called after download, before patching
- Add `_run_post_patch_engines(context)` method — called after patching, before signing
- Add `_resolve_engine_config(app_config, global_config)` helper to merge global/per-app settings

### Phase 2: High-Priority Engines

#### 2.1 Media Optimizer Engine
- **New file**: `scripts/builder/engines/media_optimizer.py`
- **Port from**: `rvp/engines/media_optimizer.py` (18KB)
- **Adaptations**:
  - Replace `Context` with `EngineContext`
  - Use Revanced-auto's logging (`from scripts.lib import logging as log`)
  - Use `scripts.utils.process` for subprocess execution instead of raw `subprocess.run`
  - Add dependency checking via `shutil.which()` with graceful degradation
- **Features ported**:
  - PNG compression via `pngquant` (lossy, 65-80% quality)
  - JPEG optimization via `jpegoptim` (85% quality, metadata stripped)
  - MP3/OGG audio re-encoding via `ffmpeg` (96k bitrate)
  - DPI-aware resource filtering (ldpi through xxxhdpi, tvdpi, nodpi)
  - Threaded parallel processing with `ThreadPoolExecutor`
  - APK extract/repack via `unzip` CLI or Python `zipfile` fallback
- **Dependencies to add**: None (external tools: `pngquant`, `jpegoptim`, `ffmpeg` — optional, detected at runtime)
- **Tests**: `tests/test_media_optimizer.py`

#### 2.2 APK Optimizer Engine
- **New file**: `scripts/builder/engines/apk_optimizer.py`
- **Port from**: `rvp/engines/optimizer.py` (11KB)
- **Adaptations**: Same pattern as 2.1
- **Features ported**:
  - Debug symbol removal (proguard, debug dirs, .map files, .log files)
  - Native library stripping via `strip --strip-unneeded`
  - AndroidManifest.xml comment removal (text manifests only, skips binary AXML)
  - Resource cleanup (.DS_Store, backup files)
  - Locale-specific resource filtering (keep only specified locales)
- **Note**: Revanced-auto already has `AAPT2Manager` for density/arch/language filtering. The APK Optimizer complements it with debug stripping and manifest minimization.
- **Tests**: `tests/test_apk_optimizer.py`

### Phase 3: Medium-Priority Engines

#### 3.1 String Cleaner Engine
- **New file**: `scripts/builder/engines/string_cleaner.py`
- **Port from**: `rvp/engines/string_cleaner.py` (9.8KB)
- **Features ported**:
  - Decompile APK via `apktool` to get plain XML
  - Scan all `strings.xml` files for defined string resources
  - Scan all `.xml` and `.smali` files for `R.string.*` and `@string/*` references
  - Remove unused string definitions from XML
  - Recompile APK via `apktool`
  - Zipalign after recompile
- **Dependencies**: `apktool` (external, runtime-detected)
- **Tests**: `tests/test_string_cleaner.py`

#### 3.2 DTL-X Engine
- **New file**: `scripts/builder/engines/dtlx.py`
- **Port from**: `rvp/engines/dtlx.py` (11KB)
- **Features ported**:
  - Full DTL-X flag mapping (30+ flags): ad removal (rmads1-5), tracker removal, SSL bypass, copy protection removal, VPN detection removal, root/Xposed checks removal, Pairip bypass, code optimization (rmnop, rmnown), manifest modifications
  - Analysis mode (decompile and report)
  - Optimization mode (apply selected flags)
  - Default optimization preset
  - Report generation
- **Dependencies**: DTL-X Python package (external, runtime-detected)
- **Tests**: `tests/test_dtlx.py`

#### 3.3 LSPatch Engine
- **New file**: `scripts/builder/engines/lspatch.py`
- **Port from**: `rvp/engines/lspatch.py` (5.8KB)
- **Features ported**:
  - Binary CLI mode (preferred) and JAR-based fallback
  - Module embedding (`-m` flag)
  - Manager mode for external LSPatch support
  - **Dual-mode integration** (configurable per app):
    - **Complement mode** (default): LSPatch runs as pre-patch engine, then ReVanced patches on top
    - **Alternative mode**: LSPatch replaces ReVanced patching entirely for that app
- **Config**:
  ```toml
  [YouTube-Morphed]
  enable-lspatch = true
  lspatch-mode = "complement"  # or "alternative"
  
  [YouTube-Morphed.lspatch]
  modules = ["module1.apk", "module2.apk"]
  manager-mode = false
  ```
- **Tests**: `tests/test_lspatch.py`

#### 3.4 Plugin System
- **New file**: `scripts/lib/plugins.py`
- **Port from**: `rvp/core.py` (discovery), `rvp/plugins/` (hook dispatch)
- **Features ported**:
  - Auto-discovery of plugin modules via `pkgutil.iter_modules()`
  - Hook stages: `pre_pipeline`, `post_pipeline`, `pre_engine:{name}`, `post_engine:{name}`
  - Plugin handler signature: `handle_hook(ctx: EngineContext, stage: str) -> None`
  - Error isolation (plugin failures don't crash pipeline)
  - Thread-safe singleton cache for discovered plugins
- **Plugin directory**: `scripts/plugins/` (auto-discovered)
- **Tests**: `tests/test_plugins.py`

## Key Design Decisions

1. **Opt-in by default**: All engines disabled by default. Existing builds are unaffected.
2. **Per-app override**: Global config sets defaults; per-app sections override.
3. **Runtime dependency detection**: External tools (pngquant, ffmpeg, apktool, etc.) are detected at runtime. Missing tools produce warnings, not crashes.
4. **No new Python dependencies**: All ported engines use external CLI tools, not Python packages. This keeps `pyproject.toml` unchanged.
5. **Existing utils reuse**: Use `scripts.utils.process` for subprocess execution, `scripts.utils.apk` for signing/alignment, existing logging infrastructure.
6. **Code style alignment**: All ported code reformatted to match Revanced-auto conventions (line-length=120, ruff ALL, mypy strict).
7. **Engine isolation**: Each engine is a self-contained module with a single `run()` entry point. Engines don't depend on each other.
8. **Plugin system included**: Auto-discovered hook-based plugin system is part of this integration (Phase 3.4), enabling future extensibility without code changes.
9. **LSPatch dual-mode**: LSPatch supports both complement mode (runs before ReVanced) and alternative mode (replaces ReVanced), configurable per app via `lspatch-mode`.

## Risk Assessment

| Risk | Mitigation |
|------|-----------|
| apktool dependency for string_cleaner | Runtime detection, graceful skip if missing |
| ffmpeg/pngquant/jpegoptim availability | Runtime detection, clear install instructions in logs |
| DTL-X compatibility | Pin to known commit, wrap in try/except |
| Pipeline slowdown from engines | Engines are opt-in; parallel file processing within engines |
| Config complexity | Sensible defaults, per-app overrides only when needed |
| Breaking existing builds | All features opt-in, no changes to default pipeline |
